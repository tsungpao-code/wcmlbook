# Exercise 2.5: DeepMIMO Channel Estimation with GAN

This repository provides the starter code for Exercise 2.5. In this exercise, you will extend the GAN-based channel modeling framework from Exercise 2.4 to work with realistic channel data generated by the **DeepMIMO** dataset (Scenario 'O1_60').

## Experiment Setup

Instead of a generic Rayleigh channel, we use the DeepMIMO dataset to generate site-specific channel coefficients.

*   **Scenario:** O1_60 (Outdoor scenario at 60 GHz)
*   **Source:** [DeepMIMO.net](https://www.deepmimo.net/)
*   **DeepMIMO Configuration:**
    *   Active BS: 1
    *   Active Users: Rows 1-500
    *   Antennas: BS 8x8 UPA, UE Single Antenna
    *   Frequency: 60 GHz

## What You Need to Do

| Checklist | Details |
| :--- | :--- |
| **Install** | Install the DeepMIMO python package: `pip install deepmimo==2.0.0` |
| **Download** | Download the 'O1_60' scenario from the DeepMIMO website and place it in `./deepmimo_scenarios/O1_60`. |
| **Code** | Open `Exercise_2.5_starter.py`. Complete `generate_deepmimo_data` to load the scenario and configure it (BS 1, Users 1-500, SISO etc.) using the v2 API parameters. |
| **Run** | Execute: `python Exercise_2.5_starter.py` |

## Key Differences from Exercise 2.4

In Exercise 2.4, the channel distribution was a mathematical Rayleigh model. Here, you are using the **DeepMIMO** ray-tracing engine to generate realistic channel data.

The GAN training logic (`generate_real_samples_with_labels_DeepMIMO`) is pre-implemented for you, as it is identical to Exercise 2.4. Your focus is on the data acquisition step.
